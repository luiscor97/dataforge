# Gobernanza

Modelo inicial (RFC-0001 §43): **benevolent maintainer**.

## Roles

- **Maintainer fundador:** Luis Cordero (Luiscor IT Services). Mantiene la
  dirección de producto y la última palabra en conflictos.
- **Maintainers:** se incorporan por historial de contribución, calidad de
  revisión, conocimiento del dominio y conducta. Pueden aprobar y fusionar
  PRs en sus áreas.
- **Contribuyentes:** cualquiera, vía PR con DCO.

## Toma de decisiones

- Cambios menores (implementación, librerías, organización): PR + ADR si
  dejan huella arquitectónica.
- Cambios mayores (garantías, formatos, licencia, lenguaje, servicios
  externos, borrado): requieren RFC aprobada. Lista completa en RFC-0001 §50.
- El roadmap no se modifica informalmente.

## Principios no negociables

Las 20 reglas absolutas de RFC-0001 §0 solo pueden cambiar mediante RFC con
aprobación del maintainer fundador.

## Código de conducta

[Contributor Covenant](CODE_OF_CONDUCT.md). Las violaciones se reportan al
correo indicado en SECURITY.md.
