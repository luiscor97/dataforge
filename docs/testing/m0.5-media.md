# M0.5 — Evidencia de pruebas de inteligencia multimedia

## Suites focales

- `df-media` (unidad): límites absolutos, contrato `automatic_action`
  irrepresentable, framing del protocolo del worker, sidecars relativos
  rechazados sin lookup de `PATH`, pHash estable.
- `df-media-worker/tests/media_integration.rs`: imagen redimensionada y
  recomprimida se relaciona y una distinta no; WAV vs audio transcodificado
  comparten Chromaprint real; vídeo generado vs recomprimido comparte
  keyframes muestreados; medios malformados fallan sin interpretarse en el
  proceso principal; la playlist no puede escapar del allowlist de pipes.
  Audio/vídeo requieren un FFmpeg local y se omiten con aviso si falta.
- `df-media-worker/tests/project_pipeline.rs` (E2E real): create → scan →
  hash → analyze → media → report con el worker aislado; rediciones JPEG
  relacionadas, run sellado y reutilizado por digest, ledger verificado, y
  fallo cerrado con evidencia explícita cuando no hay workers.
- UI: sección M0.5 pendiente hasta completar el análisis estructural,
  evidencia sellada renderizada sin acción automática (node --test).

## Cómo ejecutar

```powershell
cargo test -p df-media -p df-media-worker
pnpm --filter dataforge-desktop test:ui
```

La CLI equivalente:

```powershell
dataforge media --path <proyecto> [--ffmpeg C:\ruta\ffmpeg.exe] [--max-pairs 100000]
dataforge report media --path <proyecto>
```
