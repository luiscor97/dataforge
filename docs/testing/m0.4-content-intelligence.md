# Validación de M0.4 — inteligencia documental

Fecha local: 2026-07-18. Plataforma principal: Windows. Este documento registra
evidencia reproducible del árbol local; no afirma CI remota, commit, tag ni
release.

## Contrato probado

| Criterio RFC-0001 M0.4 | Evidencia automatizada |
| --- | --- |
| PDF/DOCX/TXT/HTML/EML/ZIP | Fixtures deterministas de `df-extract`; el parser PDF solo existe en `df-extract-worker` y se prueba por protocolo y API cliente. |
| Texto normalizado | Normalización Unicode/saltos/segmentos y hashes repetibles, con límites de caracteres y bytes. |
| Adjuntos e hilos básicos | EML conserva metadata/adjuntos; References/In-Reply-To preceden al fallback de asunto y producen IDs/orden estables. |
| ZIP seguro | Corpus hostil para traversal, cifrado, ratios, tamaños, CRC, conteo y nesting; ninguna escritura de entrada. |
| Búsqueda con ruta/contexto | E2E de facade construye Tantivy desde evidencia sellada y busca campos de ruta, contexto y texto con snippets acotados. |
| SQL analítico | Parquet contiene metadata/ruta/contexto sin texto completo; DataFusion rechaza DDL y funciones desactivadas, limita resultados y se ejecuta en `df-query-worker`. |
| Índices reconstruibles | Cada build crea artefactos inmutables nuevos y los registra solo para un run `COMPLETED`; SQLite sigue siendo la fuente. |
| Integridad check/use | Tests de leases bloquean mutación/sustitución; Tantivy y Parquet se hashean desde handles retenidos hasta terminar el consumo. |
| Reanudación/reutilización | El E2E extract→replay demuestra el mismo run sellado y reutilización cross-snapshot por contenido+versión+configuración. |
| Clientes coherentes | CLI, facade y desktop ejercitan los mismos comandos; UI cubre loading/error/success y no contiene decisiones del motor. |

## Puertas focales

```powershell
cargo test -p df-extract -p df-extract-worker -- --nocapture
cargo test -p df-db extraction::tests -- --nocapture
cargo test -p df-fs-safety -p df-search -p df-query -p df-query-worker -- --nocapture
cargo test -p df-facade content_intelligence_end_to_end -- --nocapture
cargo check -p dataforge-cli -p dataforge-desktop --all-targets
cargo clippy -p df-extract -p df-extract-worker -p df-process-safety `
  -p df-search -p df-query -p df-query-worker -p df-facade `
  -p dataforge-cli -p dataforge-desktop --all-targets -- -D warnings
pnpm --dir apps/desktop typecheck
pnpm --dir apps/desktop test:ui
pnpm --dir apps/desktop build
```

Resultado local sobre el incremento congelado: **257 pruebas Rust superadas,
0 fallidas y 0 ignoradas** en los paquetes enumerados; **7/7** pruebas
Vitest/RTL y **4/4** pruebas heredadas de render estático. `fmt` focal, Clippy
estricto, Prettier, ESLint, TypeScript y Vite están verdes. El linker GNU de la
prueba Tauri mantiene el aviso conocido de manifests `.rsrc`, con exit code 0.

La comprobación focal confirmó además que `lopdf` no aparece en el árbol de
`df-extract` y solo aparece en `df-extract-worker`. No se ha ejecutado CI
remota ni se ha creado commit, tag o release.
