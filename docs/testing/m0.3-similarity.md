# Validación de M0.3 — similitud y versionado

Fecha local: 2026-07-18. Plataforma: Windows. M0.3 se validó inicialmente con
Rust 1.82+; el workspace requiere Rust 1.88+ desde M0.4 para mantener el parser
PDF en una versión corregida frente a RUSTSEC-2026-0187 y reflejar el mínimo
real comprobado del grafo de dependencias.

Este documento registra evidencia reproducible; no sustituye CI remota ni
afirma una release/tag que no exista.

## Contrato probado

| Criterio RFC-0001 M0.3 | Evidencia |
| --- | --- |
| Variantes sintéticas relacionadas | `synthetic_versions_are_related_but_never_identical`: dos binarios de 768 KiB con una edición interna producen una única relación; un tercero independiente queda fuera. |
| Similitud distinta de identidad | La relación tiene score `> 0.30` y `< 1.0`; su JSON conserva `identity_rule=sha256_equality_only` y `automatic_action=false`. |
| Umbrales configurables | Un segundo digest con umbral `0.9999` crea otro run y cero relaciones. Se elimina el origen antes del replay para demostrar que reutiliza chunks/firmas sin releerlo. CLI acepta umbral, mínimos de evidencia y techo de candidatos; outcome, status y JSON exponen la configuración exacta y su digest. |
| Reanudación | `cancelled_run_resumes_from_the_same_configuration_identity` deja `RUNNING` y completa después el mismo `run_id`. |
| Límites explícitos | `candidate_cap_is_exactly_signalled_and_persisted` genera más de un par con máximo 1, conserva exactamente uno y sella `candidate_cap_reached=true`. |
| Persistencia acotada/sellada | `dropped_content_writer_rolls_back_every_membership` prueba publicación atómica por contenido; `content_writer_rejects_chunks_past_the_canonical_size` prueba los límites de offset/tamaño; `completed_similarity_run_seals_candidates_relations_and_run` comprueba la evidencia LSH completa y bloquea mutaciones posteriores. No se guardan bytes de archivos. |
| Clientes coherentes | La prueba de fachada recorre create→scan→hash→analyze→similarity y valida status/report; la prueba UI renderiza relación, score y advertencia no destructiva. |

## Puertas focales

```powershell
cargo test -p df-db similarity::tests -- --nocapture
cargo test -p df-similarity -- --nocapture
cargo test -p df-facade similarity_is_available_through_facade_status_and_report -- --nocapture
cargo check -p dataforge-cli --all-targets
cargo check -p dataforge-desktop --all-targets
pnpm --dir apps/desktop typecheck
pnpm --dir apps/desktop test:ui
```

Resultado local: todas verdes. La suite `df-similarity` ejecuta 5 pruebas y
mantiene un benchmark manual ignorado; las 3 pruebas focales de persistencia y
las 4 pruebas UI también pasan.

## Benchmark streaming manual

```powershell
cargo test -p df-similarity `
  benchmark_streaming_memory_is_independent_of_corpus_size `
  -- --ignored --nocapture
```

Resultado observado:

```text
[similarity benchmark] 268435456 bytes, 1024 chunks in 2.57s;
bounded working set <= two max chunks + 1024 signature bytes
```

El benchmark usa un `Read` de 256 MiB, no un `Vec` de 256 MiB. La cota de
proceso del algoritmo es el buffer interno FastCDC + el `ChunkData` actual
(dos ventanas máximas de 256 KiB con la configuración por defecto), 128 `u64`
de MinHash y páginas fijas de 64 contenidos/128 candidatos. Corpus, candidatos
y joins permanecen en SQLite.

## Comprobaciones globales

Ejecutadas sobre el árbol congelado local:

```powershell
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace -- --nocapture
cargo build -p dataforge-cli
cargo check -p dataforge-desktop
pnpm --dir apps/desktop build
cargo audit
cargo deny check
pnpm --dir apps/desktop audit --prod
git diff --check
```

- `cargo fmt --all --check`: verde.
- Clippy estricto del workspace: verde, cero warnings propios.
- Suite workspace: **312 superadas, 0 fallidas, 2 ignoradas
  intencionadamente** (escala 100k y benchmark de 256 MiB). El binario de
  pruebas Tauri conserva el aviso conocido de MinGW sobre manifests de
  recursos, con exit code 0.
- Build CLI, check Tauri y build TypeScript/Vite: verdes; UI: 4/4.
- `cargo audit`: sin vulnerabilidades rechazadas; 17 avisos permitidos ya
  inventariados de GTK3/GLib/UNIC transitivos de Tauri.
- `cargo deny check`: advisories, bans, licenses y sources verdes; solo avisos
  informativos de duplicados y allowances no encontrados.
- `pnpm audit --prod`: cero vulnerabilidades conocidas.
- `git diff --check`: verde.

M0.2 conserva por separado su evidencia del corpus final de 100.000 archivos.
No se ha afirmado CI remota, commit, tag ni release.
