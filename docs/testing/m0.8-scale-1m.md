# M0.8 — Evidencia de escala: 1.000.000 de archivos

**Fecha:** 2026-07-19 · **Máquina:** Windows 11, disco local NVMe ·
**Binario:** `scale_pipeline` en release, motor tras la optimización WAL +
hash por lotes (migración 0015).

## Ejecución

`DF_CORPUS_FILES=1000000`, corpus determinista (semilla 42, 20% duplicados).

| Fase | Resultado | Tiempo |
| --- | --- | --- |
| Generación | 1.000.000 archivos, 53.432 carpetas, 4.259.269.181 bytes | 548,7 s |
| Huella del origen (SHA-256 completo) | 1.053.432 entradas | 3.831,2 s |
| Escaneo | 1.000.000 archivos, 53.433 carpetas, **0 errores** | 267,5 s |
| Hash (SHA-256 + BLAKE3) | 1.000.000, **0 fallos** | 2.873,5 s |
| Análisis estructural | 160.147 conjuntos duplicados, 0 anomalías | 192,5 s |
| Plan | 1.093.705 operaciones (1.000.000 copias) | 184,6 s |
| **Ejecución** | **1.093.705/1.093.705 al primer intento, 0 retryable, 0 fallos finales** | 18.350,2 s |
| **Verificación independiente** | **1.093.705 comprobadas, veredicto `COMPLETED`** | 4.179,0 s |

La ejecución usa el protocolo íntegro por archivo: fingerprint pre/post,
copia en streaming con doble hash, `sync_all`, finalize sin reemplazo y
lease con token + identidad física. Cero reintentos a esta escala.

## Interrupción y comprobación post-hoc del origen

El equipo se reinició (apagado de máquina, no del motor) durante la última
aserción del test — la re-huella SHA-256 del origen — tras completarse la
verificación independiente. El test no llegó a imprimir su `ok` final.

Comprobación post-hoc sobre el árbol superviviente, tras el pipeline
completo y el reinicio:

| | Generado | Post-reinicio |
| --- | --- | --- |
| Archivos | 1.000.000 | 1.000.000 |
| Carpetas | 53.432 | 53.432 |
| Bytes totales | 4.259.269.181 | 4.259.269.181 |

Coincidencia exacta. Además, la verificación independiente (`COMPLETED`)
ya había re-hasheado cada destino contra los SHA-256 del plan aprobado,
que son los hashes del contenido original: cualquier mutación del origen
anterior a la copia habría fallado el fingerprint pre/post del executor, y
cualquier divergencia del resultado habría fallado el verifier.

**Límite declarado:** la comprobación post-hoc es por recuento, carpetas y
bytes totales, no una re-huella SHA-256 completa del origen. Un run sin
interrupciones la incluiría; puede repetirse con
`DF_CORPUS_FILES=1000000 cargo test -p df-corpus --release -- --ignored scale --nocapture`
(~8 h en esta máquina).

## Lectura

- El criterio "1M+ entradas medido" de la matriz 1.0 queda demostrado:
  1.093.705 operaciones ejecutadas y verificadas de extremo a extremo.
- El coste está donde debe: hash y ejecución son IO-bound con memoria
  acotada (el proceso nunca superó ~250 MB de working set).
- La fase dominante (ejecución, 5h06m) paga la durabilidad: `sync_all` por
  archivo antes del rename. Es el precio explícito de la garantía §27.
