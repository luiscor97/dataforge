**Estado:** activa  
**Fuente:** RFC-0001 §45 y Definition of Done general  
**Regla:** una casilla solo pasa a completada cuando existe código, prueba del
criterio completo y evidencia de la puerta de calidad. Una release/tag remoto
no se presupone a partir de cambios locales.

## Evidencia transversal exigida por cada hito

1. Migración append-only con checksum, compatibilidad desde bases anteriores y
   pruebas de constraints/triggers.
2. Contratos de dominio y JSON versionados, sin reinterpretar evidencia
   histórica.
3. Algoritmos deterministas, acotados y con configuración incluida en la
   evidencia que producen.
4. Fallo cerrado ante fuente modificada, formato hostil, límite agotado o
   plataforma sin garantía equivalente.
5. CLI, facade y escritorio consumen el mismo motor; la UI no decide.
6. `fmt`, clippy estricto, tests, builds, auditorías de dependencias y pruebas
   end-to-end verdes.
7. CHANGELOG, README, ADR, arquitectura y threat model actualizados antes de
   declarar completado el incremento.
